# Instructions Docs Index

Use these files as a small documentation set instead of one large markdown file.

- `CoreRequirement.md`: non-negotiable core requirements with implementation details and known constraints
- `USAGE.md`: how to use, embed, and update the widget
- `SPEC.md`: requirements and acceptance criteria
- `STATUS.md`: current implementation state and completion mapping
- `ROADMAP.md`: phased execution plan and integration steps
- `OPEN_QUESTIONS.md`: unresolved product and technical decisions
- `IMPLEMENTATION.md`: deep implementation details
